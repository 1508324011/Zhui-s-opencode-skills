# 思维框架提炼方法论

> 本文件保留上游方法论，并去掉了对 Claude 宿主的依赖假设。它只负责回答一个问题：什么样的观点值得被提炼为可运行的心智模型。

## 一、心智模型识别的三重验证

一个论点要被认定为「心智模型」而非「随口一说」，必须通过三重验证。

### 验证 1：跨域复现

同一个思维框架出现在此人讨论的至少 2 个不同领域。

例：纳瓦尔的“杠杆”概念——

- 在财富创造中谈杠杆
- 在个人成长中谈杠杆
- 在职业选择中谈杠杆

跨多个场景复现，说明它不是偶发表达，而是底层镜片。

### 验证 2：有生成力

用这个模型可以推断此人对新问题的可能立场。

例：如果芒格的“逆向思维”是核心模型，那么面对“如何成功”，他会先想“如何避免失败”；面对“如何投资”，他会先问“什么会让我永久亏钱”。

能把旧观点转化成对新问题的判断，才说明模型是可运行的。

### 验证 3：有排他性

这个模型不是“所有聪明人都这么想”，而是此人稳定、独特、可辨认的视角。

例：“反脆弱”是塔勒布式的；“欲望是跟不快乐签合同”是 Naval 式的；“端到端控制”是乔布斯式的。

如果一个观点只通过了 1 重验证，把它降级为决策启发式；如果 3 重都过不了，就不要纳入。

## 二、表达 DNA 的提取方法

表达 DNA 不是模仿口头禅，而是识别这个人怎么组织判断、怎么施加语气、怎么控制节奏。

建议从以下维度提取：

| 维度 | 要看什么 |
| --- | --- |
| 句式 | 长句/短句、疑问/陈述、先结论还是先铺垫 |
| 词汇 | 高频词、自创术语、偏爱或避开的词 |
| 节奏 | 转折频率、停顿方式、是否层层推进 |
| 幽默 | 讽刺、自嘲、荒诞、冷幽默或完全不幽默 |
| 确定性 | 是强断言型还是谨慎限定型 |
| 引用习惯 | 爱引用谁、爱引用什么类型的材料 |

目标是让生成物“有辨识度但不滑向 caricature”。

## 三、矛盾处理原则

矛盾不是噪声，往往正是人物深度的来源。

### 1. 时间性矛盾

此人早期说 A，后期说 B。

处理方式：保留演化轨迹，标明时间，不要把它伪装成一以贯之。

### 2. 场景性矛盾

同一人面对不同领域使用不同规则。

处理方式：分场景记录，不强行统一。

### 3. 本质性张力

例如既极度重视自由，又对纪律有强依赖；既要求极简，又追求极致控制。

处理方式：在“价值观与反模式”或“诚实边界”中显式保留张力。

错误做法：

- 选一边、忽略另一边
- 编一个没有证据的调和故事
- 假装矛盾不存在

## 四、信息不足时怎么处理

| 情况 | 处理方式 |
| --- | --- |
| 某个维度公开信息稀缺 | 在成品中标注“信息不足，此维度为弱推断” |
| 只有二手信息 | 降低置信度，明确标为二手来源 |
| 信息互相矛盾 | 并列保留，不强行裁决 |
| 此人刻意回避某个话题 | 把沉默本身视为边界，不替他填空 |

## 五、人物 skill 与主题 skill 的区别

| 维度 | 人物 skill | 主题 skill |
| --- | --- | --- |
| 核心对象 | 一个人的思维方式 | 一个领域的方法论与分歧 |
| 心智模型来源 | 主要来自一个人 | 综合多人或多个流派 |
| 表达风格 | 默认保留人物视角 | 默认使用中性、专业表达 |
| 张力来源 | 人物内在矛盾 | 流派之间的根本分歧 |

## 六、质量自检清单

在生成最终 `SKILL.md` 之前，自查以下问题：

### 心智模型

- [ ] 每个模型都有至少 2 个不同场景的支撑
- [ ] 模型数量在 3-7 个之间
- [ ] 每个模型都说明了应用方式和局限
- [ ] 模型之间既有联系，也保留必要张力

### 表达 DNA

- [ ] 读起来有辨识度，不像通用 AI 文风
- [ ] 不是表面模仿，不是堆口头禅
- [ ] 能解释“为什么这像这个人”

### 决策启发式

- [ ] 每条启发式都有具体案例或明确来源
- [ ] 面对新问题时有复用价值

### 诚实边界

- [ ] 明确写出做不到什么
- [ ] 明确标注调研时间与信息截止点
- [ ] 承认信息不足与推断成分

### 整体

- [ ] 这个 skill 能帮助用户“用对方的镜片看问题”，而不是复读语录
- [ ] 删掉名字后，仍能从风格和判断中看出是谁
- [ ] 如果依赖最新事实，下游 skill 知道应该先研究再回答
